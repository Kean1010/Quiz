LocalStorage Test

About the Issue

The "Invalid game code" error occurs because GitHub Pages serves your website from a different origin than where you created the quiz. LocalStorage is origin-specific, meaning data stored on one origin (e.g., your local computer) cannot be accessed from another origin (e.g., GitHub Pages).

LocalStorage is not shared between different origins. This is a browser security feature.

Solutions

To make your quiz app work across different devices and origins, you need a way to share data between them. Here are some options:

Option 1: Use a Backend Service

The most robust solution is to use a backend service to store quiz data. This could be a simple API built with Node.js, Firebase, or any other backend technology.

Option 2: Use URL Parameters

For a simple solution, you could encode quiz data in URL parameters. This works for small quizzes but has limitations on data size.

Option 3: Use IndexedDB with Sync

For a more advanced solution, you could use IndexedDB with a sync mechanism to share data between devices.

Quick Fix for Testing

For testing purposes, you can create and join quizzes on the same device and browser:

  1. Open your GitHub Pages website in a browser
  2. Create a new quiz
  3. Copy the game code
  4. Open a new tab in the same browser
  5. Join the quiz using the copied code

This works because both tabs share the same origin (GitHub Pages) and can access the same LocalStorage data.
